# KLEP decisions

This log separates direct owner rules from rewrite choices that still need the
owner's visual/semantic approval. `Accepted` means the owner stated the rule in
this task or supplied it as the decision in the Key/Lock handoff. `Candidate`
means the rewrite implements the rule for inspection, but the owner has not yet
accepted it as permanent KLEP behavior. `Superseded` retains an earlier rule as
history after a later Accepted decision replaced it; it is no longer active.
