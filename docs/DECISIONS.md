# KLEP decisions

This log separates direct owner rules from rewrite choices that still need the
owner's visual/semantic approval. `Accepted` means the owner stated the rule in
this task or supplied it as the decision in the Key/Lock handoff. `Candidate`
means the rewrite implements the rule for inspection, but the owner has not yet
accepted it as permanent KLEP behavior. `Superseded` retains an earlier rule as
history after a later Accepted decision replaced it; it is no longer active.

## Still open

- Whether emitted occurrences may always override a definition's default lifetime.
- Reservation, use counts, borrowing and return beyond the explicit V1
  operations, and winning-candidate consumption.
- Whether untrusted agent code needs a capability layer that restricts direct
  mutation API and Neuron-handle access. V1 assumes trusted host orchestration.
- The generic stable entity-handle and Unity registry contract for legacy
  Vector/Transform/object payload uses. The zombie test uses scene-authored text
  IDs and the Enemy Sensor's explicitly referenced latest-sample map only; that
  test-scoped resolver does not settle spawned identity or general registry
  ownership.
- Whether a formal world/tick coordinator should wrap the currently explicit
  Global `KLEPKeyStore.CommitBoundary` call.
- Future score components for Executables, including whether Key attractiveness
  returns and how composite `Any` and `Not` expressions contribute without
  rewarding unrelated Keys.
- Whether Agent confidence itself may later influence a Solo switching margin.
  Observer polish is now a separate accepted score component and does not use
  confidence as its magnitude.
- The project context and policy rules for the Humans-versus-Zombies Ethics
  experiment, including the observed evidence and causal timestamps those rules
  require.
- A proposal combiner beyond KLEP-COG-002's Memory/Emotion Observer evidence,
  including Planner and Agent-learning proposals without letting any proposal
  bypass Locks.
- The project-owned file codec and atomic save boundary for `KLEPMemoryState`,
  plus whole-Agent composition with live Keys, Executable lifecycle, Emotion,
  and Agent learning.
- Open-experience persistence when logout occurs after a Prior or During moment
  but before a factual consequence has been observed.
- Multi-step Observer route search after Executables declare whether expected
  effects add, replace, or remove Keys; `DeclaredOutputs` alone is insufficient
  to infer a truthful state transition. Nora/Aron competition and its
  trust/resource-allocation rules remain deferred.
- Whether an unchanged low-confidence episode may also reconsult on an explicit
  external evidence revision or deterministic cooldown. Payload changes in
  visible Keys are now covered by KLEP-OBS-009; a Memory, Emotion, or project
  evidence change not represented in those Keys still has no revision seam.
- Whether a holistic direction remains a fixed-strength nudge that may lose
  ordinary arbitration, or should receive enough bounded influence to guarantee
  selection among currently eligible targets.
- Global same-world-tick influence across several Neurons; this requires an
  explicit world coordinator rather than a Neuron-owned tandem wave.
